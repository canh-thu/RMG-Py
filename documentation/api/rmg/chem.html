2.2. The rmg.chem Module¶

Contains classes describing simple chemical entities:

  • Element - A chemical element
  • AtomType - A type of atom, combining element information with local bonding information
  • ElectronState - A free electron state for an atom, describing the number of free electrons and the spin multiplicity
  • BondType - A type of bond, reflecting the bond order
  • Atom - A chemical atom, combining one (or more) atom types with one (or more) electron states
  • Bond - A chemical bond, combining one (or more) bond types

2.2.1. Classes in rmg.chem¶

2.2.1.1. Element¶

class rmg.chem.Element¶

A single chemical element. The attributes are:

Attribute Description
number The atomic number of the element
name The name of the element
symbol The symbol used for the element
mass The mass of the element in kg/mol
valence A list of the allowed numbers of bonds to this element

This class is specifically for properties that all atoms of the same element share. Ideally there is only one instance of this class for each element.

The elements available in RMG, which can be accessed by atomic number or symbol using the elements dictionary in this module, are:

Number Symbol Element
0 R a generic element
1 H hydrogen
2 He helium
6 C carbon
7 N nitrogen
8 O oxygen
9 F fluorine
10 Ne neon
14 Si silicon
15 P phosphorous
16 S sulfur
17 Cl chlorine
18 Ar argon
35 Br bromine
53 I iodine

2.2.1.2. AtomType¶

class rmg.chem.AtomType¶

A type of atom which combines the element information with information about the local bonding structure. The attributes are:

Attribute Description
label A unique string identifier for the atom type
element The Element object this atom type represents, if any
description A one-line description of the atom type
doubleBonds The number of double bonds to this atom type
tripleBonds The number of triple bonds to this atom type
benzeneBonds The number of benzene bonds to this atom type
formBond The atom type(s) resulting from an adjacent bond formation
breakBond The atom type(s) resulting from an adjacent bond breaking
incrementBond The atom type(s) resulting from an adjacent bond order increment
decrementBond The atom type(s) resulting from an adjacent bond order decrement

This class is specifically for properties that all atoms of the same type share. Ideally there is only one instance of this class for each atom type.

equivalent()¶
Returns True if two atom types are equivalent or False otherwise. Respects wildcards, e.g. returns True for {R!H}=={C}
setActions()¶

Set the atom types that result from the application of the relevant chemical actions. For atom types, these actions are:

  • FORM_BOND - form a single bond between this atom and another atom
  • BREAK_BOND - break a single bond between this atom and another atom
  • INCREMENT_BOND - increment the order of a bond involving this atom by one
  • DECREMENT_BOND - decrement the order of a bond involving this atom by one

If an action cannot be applied to an atom type, it should be set to None. If multiple atom types can result from an action, a list should be used; RMG will eliminate atom types based on the resulting structure, if possible.

The atom types defined in RMG, which can be accessed by label using the atomTypes dictionary in this module, are:

Label Description
R generic functional group
R!H generic non-hydrogen functional group
H hydrogen
C carbon
N nitrogen
O oxygen
F fluorine
Ne neon
Si silicon
P phosphorus
S sulfur
Cl chlorine
Ar argon
Br bromine
I iodine
Ct carbon with one triple bond and one single bond
Cs carbon with four single bonds
Cd carbon with one double bond and two single bonds
Cdd carbon with two double bonds
Cb carbon belonging to a benzene ring
Cbf carbon belonging to a fused benzene ring
Os oxygen with two single bonds
Od oxygen with one double bond
Sit silicon with one triple bond and one single bond
Sis silicon with four single bonds
Sids silicon with one double bond and two single bonds
Sidd silicon with two double bonds
Sib silicon belonging to a benzene ring
Sibf silicon belonging to a fused benzene ring

2.2.1.3. ElectronState¶

class rmg.chem.ElectronState¶

A single free electron state for an atom. The attributes are:

Attribute Description
label A unique string identifier for the electron state
order The number of free electrons
spin A list of the allowed spin states for polyradicals (singlet = 1, doublet = 2, etc.)
increment The electron state that results from incrementing the radical count, if any
decrement The electron state that results from decrementing the radical count, if any

This class is specifically for properties that all free electron states share. Ideally there is only one instance of this class for each free electron state.

equivalent()¶
Returns True if two electron states are equivalent or False otherwise.
setActions()¶

Set the electron states that result from the application of the relevant chemical actions. For electron states, these actions are:

  • INCREMENT_RADICAL - increment the free electron count by one
  • DECREMENT_RADICAL - decrement the free electron count by one

If an action cannot be applied to an electron state, it should be set to None.

The free electron states available in RMG, which can be accessed by label using the electronStates dictionary in this module, are:

Label Order Spin multiplicities
0 0 1
1 1 1
2 2 1, 3
2S 2 1
2T 2 3
3 3 2, 4
4 4 1, 3, 5

2.2.1.4. BondType¶

class rmg.chem.BondType¶

A type of chemical bond. The attributes are:

Attribute Description
label A unique string identifier for the bond type
name A longer descriptor of the bond type
order The bond order
piElectrons The number of pi electrons that contribute to the bond
location Geometric description of bond if needed (i.e. ‘cis’ or ‘trans’)

This class is specifically for properties that all bonds of the same type share. Ideally there is only one instance of this class for each bond type.

equivalent()¶
Returns True if two bond types are equivalent or False otherwise. The method is strictly a comparison of the bond type via the label attribute. Unlike with atom types, no fuzzy matching of bond types is currently allowed (i.e. ‘D’ is different from ‘Dcis’ and ‘Dtrans’).

The bond types available in RMG, which can be accessed by label or order using the bondTypes dictionary in this module, are:

Label Name Order Pi electrons Location
S single 1 0 n/a
D double 2 2 n/a
Dcis double_cis 2 2 cis
Dtrans double_trans 2 2 trans
T triple 3 4 n/a
B benzene 1.5 1 n/a

2.2.1.5. Atom¶

class rmg.chem.Atom¶

Represent an atom in a chemical species or functional group. The attributes are:

Attribute Description
atomType The atom’s type or a list of the allowed types
electronState The atom’s free electron state or a list of the allowed states
charge The formal charge of the atom (always zero)
label A string used to tag individual atoms, e.g. center atoms or reactive sites
atomType¶
Get the list of allowed atom types. If the list is of length 1, the lone item in the list is returned instead.
breakBond()¶
Modify the atom type(s) of this atom in response to a BREAK_BOND chemical action.
canDecreaseFreeElectron()¶
Return True if the number of unpaired electrons on this atom can be decreased by one. This method requires that there be at least one free electron on this atom.
canIncreaseFreeElectron()¶
Return True if the number of unpaired electrons on this atom can be increased by one. This method restricts the radical order to three or fewer.
copy()¶
Generate a copy of the current atom.
decreaseFreeElectron()¶
Decrease the number of unpaired electrons on this atom by one.
decrementBond()¶
Modify the atom type(s) of this atom in response to a DECREMENT_BOND chemical action.
electronState¶
Get the list of allowed electronic states. If the list is of length 1, the lone item in the list is returned instead.
equals()¶
Equality comparison.
equivalent()¶
Return True if self and other are equivalent atoms, i.e. they have the same atom type and electronic state. If one or both of the atoms being tested for equivalence have multiple atom types, True is returned if any atom type in one atom matches any atom type in the other. This is also done for electron states. Both an atom types match and an electron state match must be found in order to return True.
formBond()¶
Modify the atom type(s) of this atom in response to a FORM_BOND chemical action.
getAtomType()¶
Get the list of allowed atom types. If the list is of length 1, the lone item in the list is returned instead.
getElectronState()¶
Get the list of allowed electronic states. If the list is of length 1, the lone item in the list is returned instead.
getElement()¶
Return the element that this atom represents. If there are multiple atom types, then this method returns None.
getFreeElectronCount()¶
Return the number of unpaired electrons.
hasFreeElectron()¶
Return True if the atom has one or more unpaired electrons and False otherwise.
increaseFreeElectron()¶
Increase the number of unpaired electrons on this atom by one.
incrementBond()¶
Modify the atom type(s) of this atom in response to a INCREMENT_BOND chemical action.
isCarbon()¶
Return True if the atom is a carbon atom and False otherwise.
isCenter()¶
Return True if the atom is a center atom and False otherwise. An atom is a center atom if it has any form of label.
isElement()¶
Return True if the atom has the element with the symbol symbol and False otherwise. Returns False if atom is a wildcard (e.g. in a functional group)
isHydrogen()¶
Return True if the atom is a hydrogen atom and False otherwise.
isNonHydrogen()¶
Return True if the atom is not a hydrogen atom and False otherwise.
isOxygen()¶
Return True if the atom is an oxygen atom and False otherwise.
setAtomType()¶

Set the atom type that this atom represents. The atomType parameter is any of:

  • A string containing the label of a single atom type
  • An AtomType object representing the atom type
  • A list containing one or more of each of the above

In all cases, the data will be stored internally as a list of AtomType objects.

setElectronState()¶

Set the electron state that this atom represents. The electronState parameter is any of:

  • A string containing the label of a single electron state
  • An ElectronState object representing the electron state
  • A list containing one or more of each of the above

In all cases, the data will be stored internally as a list of ElectronState objects.

2.2.1.6. Bond¶

class rmg.chem.Bond¶

A chemical bond between atoms. The attributes are:

Attribute Description
atoms A list of the two atoms linked by the bond
bondType The bond’s type or a list of the allowed types
bondType¶
Get the list of allowed bond types. If the list is of length 1, the lone item in the list is returned instead.
canDecreaseOrder()¶
Return True if the bond order can be decreased by one without breaking.
canIncreaseOrder()¶
Return True if the bond order can be increased by one.
copy()¶
Generate a copy of the current bond.
decreaseOrder()¶
Decrease the bond order by one.
equals()¶
Equality comparison.
equivalent()¶
Return True if self and other are equivalent bonds, i.e. they have the same bond type. If one or both of the bonds being tested for equivalence have multiple bond types, True is returned if any bond type in one bond matches any bond type in the other.
getBondType()¶
Get the list of allowed bond types. If the list is of length 1, the lone item in the list is returned instead.
increaseOrder()¶
Increase the bond order by one.
isBenzene()¶
Return True if the bond represents a benzene (aromatic) bond and False otherwise.
isDouble()¶
Return True if the bond represents a double bond and False otherwise.
isSingle()¶
Return True if the bond represents a single bond and False otherwise.
isTriple()¶
Return True if the bond represents a triple bond and False otherwise.
setBondType()¶

Set the bond type that this atom represents. The bondType parameter is any of:

  • A number representing the bond order
  • A string containing the label of a single bond type
  • An BondType object representing the bond type
  • A list containing one or more of each of the above

In all cases, the data will be stored internally as a list of BondType objects.

2.2.2. Methods in rmg.chem¶

rmg.chem.loadAtomTypes()¶
Loads entries into a dictionary of atom types.
rmg.chem.loadBondTypes()¶
Loads entries into a dictionary of bond types. The dictionary created by this function is always available at rmg.chem.bondTypes.
rmg.chem.loadElectronStates()¶
Loads entries into a dictionary of free electron states. The dictionary created by this function is always available at rmg.chem.electronStates.
rmg.chem.loadElements()¶
Loads entries into a dictionary of elements.

2.2.3. Exceptions in rmg.chem¶

exception rmg.chem.InvalidChemicalActionException¶
An exception used attempting to carry out a chemical action (bond formation or breaking, increasing or decreasing free radical count on an atom, etc.) that is not permitted.
