12.1. IntroductionÂ¶

The following is a reference to the application programming interface (API) used within MEASURE. If you are a developer wishing to know the full implementation details of MEASURE - e.g. if you are trying to modify the codebase - this document is for you. If you are simply trying to learn how to generate phenomenological rate coefficients with MEASURE, you should switch to the MEASURE User’s Guide. If you want to know the mathematical basis for the master equation methods used by measure, you should read the MEASURE Theory Guide. (In fact, if you haven’t already, you may wish to read these documents first before continuing.)

The source for MEASURE is split across a number of Python and Cython modules. Each section of the developers’ guide references a different module:

Module Description
rmgpy.measure.input Reading network data from input files
rmgpy.measure.network Representing network data in memory
rmgpy.measure.collision Working with collision models and parameters
rmgpy.measure.reaction Working with microcanonical and phenomenological rate coefficients
rmgpy.measure.msc Determining \(k(T,P)\) via the modified strong collision method
rmgpy.measure.rs Determining \(k(T,P)\) via the reservoir state method
rmgpy.measure.output Writing network data to input and output files
rmgpy.measure.settings Setting application-wide options
